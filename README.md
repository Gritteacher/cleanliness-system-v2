# ระบบตรวจความสะอาดคณะสี V2

เว็บแอพ React + Vite แบบแยกไฟล์ รองรับ Mobile First  
เชื่อม ระบบสมาชิก / ฐานข้อมูล / พื้นที่จัดเก็บรูป แล้ว

## วิธีติดตั้ง

```bash
npm install
npm run dev
```

เปิดตาม URL ที่ Vite แสดง เช่น `http://localhost:5173`

## การตั้งค่า ระบบ

อ่านไฟล์:

```text
SUPABASE_SETUP.md
```

และรัน SQL ในโฟลเดอร์:

```text
supabase/
```

ตามลำดับ

```text
01_schema.sql
02_seed_master_data.sql
03_create_profiles_after_auth_users.sql
```

## สิ่งที่เชื่อมแล้ว

- ระบบ Auth สำหรับ Login
- ระบบ Database สำหรับข้อมูลพื้นที่ ข้อมูลการมาทำเวร คะแนนความสะอาด และประวัติแก้ไข
- ระบบ Storage สำหรับรูปภาพพื้นที่
- ลดขนาดรูปก่อนอัปโหลด
- หน้า Public อ่านข้อมูลจาก ระบบ
- ประธานบันทึกข้อมูลและคะแนนลง ระบบ
- Admin แก้ไข/ลบข้อมูลการมาทำเวรลง ระบบ
- Admin ติ๊กเลือกลบข้อมูลทดลองจากหน้าเว็บได้ เช่น เวร คะแนน Log รูป และ Cache
- ผู้ใช้เปลี่ยนชื่อผู้ใช้และรหัสผ่านได้
- Admin เห็นชื่อผู้ใช้และรหัสสำรองที่บันทึกไว้
- Admin อัปโหลดรูปแทนคณะสีได้
- Admin เข้าเมนูทำเวรแทนหัวหน้าคณะสีเจ้าของเวรของวันที่เลือกได้ทุกสี
- Admin เข้าเมนูให้คะแนนและเลือกใช้สิทธิ์แทนหัวหน้าคณะสีใดก็ได้
- Admin ให้คะแนนได้ทุกวันโดยไม่ล็อกคณะสีตามวัน สามารถเลือกคณะสีที่ต้องการตรวจเองได้
- Area Setup บันทึกพื้นที่ลง ระบบ

## อัปเดตล่าสุด

- Admin Summary เลือกช่วงสรุปได้หลายแบบ: รายวัน รายเดือน ช่วงเดือน รายปี และช่วงวันที่
- ตารางคะแนนความสะอาดและตารางคะแนนการบริหารจัดการจะเรียงอันดับ 1-5 ตามช่วงที่เลือก


## อัปเดตฟอนต์

- ปรับ Font ทั้งระบบเป็น Prompt Regular


## อัปเดตรายงานการกรอกข้อมูล

- เพิ่มหน้า Admin: รายงานการกรอกข้อมูล
- เลือกรายงานได้แบบรายวัน รายเดือน ช่วงเดือน รายปี และช่วงวันที่
- กรองคณะสีและสถานะได้
- แสดงรายละเอียดว่าแต่ละสีกรอกข้อมูลอะไรไปบ้าง เช่น ห้อง พื้นที่ สถานะ จำนวนคน รูป คะแนน ผู้กรอก และหมายเหตุ
- ห้องที่เลือกสถานะไปกิจกรรมจะไม่นำมาคำนวณทุกคะแนน และไม่ถูกนับในจำนวนห้องเรียน


## อัปเดตรายละเอียดคะแนนรายประธาน

- หน้า Admin รายงาน แสดงคะแนนที่ประธานแต่ละคณะสีให้เป็นรายคน
- แสดงเหตุผลหรือหมายเหตุการให้คะแนนของประธานแต่ละสี
- หากประธานสีใดยังไม่ให้คะแนน ระบบจะแสดงว่า “ยังไม่ให้คะแนน”


## อัปเดตหน้าสาธารณะ

- หน้าสาธารณะแสดงเหตุผลการให้คะแนนรายห้อง
- หน้าสาธารณะไม่แสดงคณะสีผู้ประเมินและไม่แสดงคะแนนรายประธาน
- หากเป็นห้องที่ไปกิจกรรม ระบบจะแสดงว่ายกเว้นการประเมินและไม่นำมาคำนวณคะแนน


## อัปเดตหน้าสาธารณะเหตุผลเท่านั้น

- หน้าสาธารณะแสดงเฉพาะเหตุผลการให้คะแนน
- ไม่แสดงว่าประธานสีใดให้คะแนน
- ไม่แสดงคะแนนรายประธาน


## อัปเดต PDF และอีเมลรายวัน

- เพิ่มปุ่มสร้าง PDF ในหน้าสาธารณะ
- ก่อนสร้าง PDF ต้องกรอกชื่อผู้ใช้และรหัสผ่านของระบบ
- เพิ่ม Netlify Scheduled Function สำหรับส่ง PDF หน้าสาธารณะทางอีเมลทุกวันเวลา 09:00 น. ตามเวลาประเทศไทย
- ตั้งค่าเพิ่มเติมตามไฟล์ `NETLIFY_PDF_EMAIL_SETUP.md`


## อัปเดตแก้ PDF Function

- แก้ `browserType.launch: headless: expected boolean, got string`
- บังคับค่า `headless: true` ใน Netlify Function สำหรับสร้าง PDF


## อัปเดตแก้ข้อความ error PDF

- ปรับให้ปุ่มสร้าง PDF แสดงสาเหตุ error จาก Netlify Function ชัดเจนขึ้น
- ปรับการรอโหลดหน้า PDF เพื่อลดโอกาส timeout จากการรอ network idle


## แก้ timeout PDF 30 วินาที

- ปุ่มสร้าง PDF แบบกดเองเปลี่ยนเป็นการยืนยันตัวตน แล้วเปิดหน้า Print/Save as PDF ใน browser
- เพิ่ม `validate-pdf-login` สำหรับตรวจ user/password แบบเร็ว
- ส่งอีเมลรายวันเปลี่ยนให้ Scheduled Function เรียก Background Function เพื่อหลีกเลี่ยง timeout 30 วินาที
